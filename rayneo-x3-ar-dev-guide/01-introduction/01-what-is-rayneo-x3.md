# RayNeo X3 硬件特性与架构说明

## 1. 设备定位

RayNeo X3 是一款透视式 AR（Augmented Reality）眼镜，运行完整的 Android 12 系统。与普通 Android 手机的核心区别在于：它将数字内容叠加在用户对真实世界的视野上，而不是显示在一块不透明的屏幕上。

理解这一点至关重要：**用户既要看到你的 UI，也要同时看到现实世界**。这决定了 AR 应用在设计和开发上的所有特殊要求。

---

## 2. 屏幕系统：双目显示架构

### 2.1 物理结构

X3 内含**两块独立的微型显示屏**：

```
┌─────────────────────────────────────────┐
│           RayNeo X3 眼镜                 │
│                                         │
│   [左眼显示屏]          [右眼显示屏]     │
│   投射到左眼             投射到右眼      │
│                                         │
│   ◄──────── 逻辑屏幕（合并宽度）────────►│
└─────────────────────────────────────────┘
```

Android 系统将这两块屏幕**合并为一块逻辑屏幕**暴露给 App。逻辑屏幕的宽度 = 左屏宽 + 右屏宽。

### 2.2 画面撕裂问题

当一个普通 Android App 运行在 X3 上时，系统会将逻辑屏幕平分：

- 左半部分的像素 → 左眼显示屏
- 右半部分的像素 → 右眼显示屏

结果：一个按钮可能左眼看到左半边、右眼看到右半边，用户会看到**撕裂感**，根本无法正常使用。

### 2.3 合目（Fusion Vision）方案

正确做法是：同一份 UI 布局，左右眼各放一份，让左右眼看到完全相同（或微调后）的内容：

```
逻辑屏幕（整宽）
┌──────────────────┬──────────────────┐
│    左眼内容       │    右眼内容       │
│  ┌────────────┐  │  ┌────────────┐  │
│  │  Hello AR  │  │  │  Hello AR  │  │
│  └────────────┘  │  └────────────┘  │
└──────────────────┴──────────────────┘
      ↓ 用户实际看到（双目融合）
         ┌────────────┐
         │  Hello AR  │  ← 清晰、无撕裂
         └────────────┘
```

Mercury SDK 的核心价值之一，就是通过 `BaseMirrorActivity`、`BindingPair` 等组件，**自动处理这个双份布局的同步问题**，让你只写一份业务逻辑。

## 3. 交互系统：镜腿触控板

X3 的眼镜框（镜腿）上内置了触控板（Touchpad）。这是目前**唯一的主要交互输入方式**。

### 3.1 物理位置

```
    镜片                               镜腿
┌──────────┐                    ┌──────────────────┐
│          │────────────────────│ ████ 触控区域 ████│
│          │   右侧眼镜腿        │                  │
└──────────┘                    └──────────────────┘
```

目前 SDK 开放监听的是**右侧镜腿**的 TP 事件（左侧镜腿技术上也可触发，但当前 SDK 只暴露右侧）。

### 3.2 支持的手势类型

| 手势 | TempleAction 类型 | 典型用途 |
|------|-------------------|---------|
| 单指单击 | `Click` | 确认/选中 |
| 单指双击 | `DoubleClick` | 返回/退出页面 |
| 单指长按 | `LongClick` | 次要操作/上下文菜单 |
| 单指三击 | `TripleClick` | 特殊快捷操作 |
| 向前（朝镜片方向）滑动 | `SlideForward` | 下一项/下一页 |
| 向后（朝镜腿方向）滑动 | `SlideBackward` | 上一项/上一页 |
| 向上滑动 | `SlideUpwards` | 垂直导航（X3新增）|
| 向下滑动 | `SlideDownwards` | 垂直导航（X3新增）|
| 连续滑动 | `SlideContinuous` | 跟手列表滚动 |
| 双指单击 | `DoubleFingerClick` | 辅助操作（X3新增）|
| 双指长按 | `DoubleFingerLongClick` | 辅助操作（X3新增）|
| 按下/抬起（原始） | `ActionDown` / `ActionUp` | 底层事件 |

> **自然模式 vs 非自然模式**：用户可在系统设置中切换。两种模式下 SlideForward/SlideBackward 的物理方向相反。你的应用应以手势语义（"前进"/"后退"）而非物理方向来理解事件。

### 3.3 附件支持

配对连接的戒指配件也会产生 TP 事件，与镜腿事件使用相同的 API。

## 4. 摄像头系统

X3 配备多个摄像头：

| 摄像头 | 用途 | 分辨率 |
|--------|------|--------|
| 主摄像头（前置） | 拍照、视频、AR 识别 | 多档，详见 Camera2 枚举 |
| VGA 摄像头 (Camera ID: 1) | 空间定位追踪 | 最高 640×480 |

VGA 摄像头是专用的空间定位传感器，开发者通常不直接操作它，但了解它的存在有助于理解 X3 的空间感知能力。

---

## 5. IMU 传感器

X3 内置完整的惯性测量单元（IMU）：

- **加速度计**：线性加速度
- **陀螺仪**：角速度
- **磁力计**：地磁方向
- **游戏旋转矢量（软件融合）**：最佳的头部姿态四元数，不依赖磁力计，无磁场干扰

IMU 数据通过标准 Android `SensorManager` API 获取，无需特殊 SDK。

---

## 6. 音频系统

- 支持蓝牙 A2DP（音乐音频）和 HFP（通话音频）
- SDK 提供 `SoundBridge` 工具类，用于在多种音频路由间切换
- X3 有特有的 MIC 收音模式，与 X2 有所不同

## 7. 系统信息速查

| 属性 | 值 |
|------|-----|
| 操作系统 | Android 12 |
| 最低目标 SDK | API 26 |
| 显示架构 | 双目独立显示，合并为逻辑屏幕 |
| 主要输入 | 右侧镜腿触控板 |
| 摄像头 | 主摄 + VGA 定位摄像头 |
| IMU | 加速度计 + 陀螺仪 + 磁力计 |
| 蓝牙音频 | A2DP / HFP |
